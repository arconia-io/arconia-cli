= `arconia project`

The `arconia project` command allows you to create new Java projects from templates and publish them as OCI artifacts. Published projects can be organized into collections for discovery and sharing.

== Commands

=== `create`

Create a new Java project from a template.

[source,shell]
----
arconia project create \
  --name my-app \
  --template ghcr.io/arconia-io/arconia-project-templates/server-http

arconia project create \
  --name my-app \
  --template server-http # Automatically resolved from a registered project collection

arconia project create \
  --name my-app \
  --template ghcr.io/arconia-io/arconia-project-templates/server-http \
  --group io.arconia \
  --description "My web application project" \
  --package-name io.arconia.app
----

**Options**

[cols="1,1,3"]
|===
|Option |Default |Description

|`--name`
|
|The name of the project to create.

|`--template`
|
|The project to use as a template (e.g. `ghcr.io/arconia-io/arconia-project-templates/server-http`).

|`--group`
|`com.example`
|The group ID for the project.

|`--description`
|
|The description for the project.

|`--package-name`
|
|The package name for the project.

|`--path`
|_(current directory)_
|The path where the project will be created.

|`--registry-insecure`
|`false`
|Call the OCI registry over HTTP instead of HTTPS.

|`--registry-skip-tls-verify`
|`false`
|Allow TLS connections without validating certificates.

|`--verbose` or `-v`
|`false`
|Include verbose output.

|`--help` or `-h`
|
|Display help information for the command.
|===

=== `push`

Publish a Java project as an OCI artifact. Exactly one of `--ref` or `--base-ref` must be specified.

[source,shell]
----
arconia project push --ref ghcr.io/arconia-io/arconia-project-templates/server-http
arconia project push --ref ghcr.io/arconia-io/arconia-project-templates/server-http --tag 1.0.0
arconia project push --base-ref ghcr.io/arconia-io/arconia-project-templates --output-report
----

**Options**

[cols="1,1,3"]
|===
|Option |Default |Description

|`--ref`
|
|The full OCI artifact reference for a single project (e.g. `ghcr.io/arconia-io/arconia-project-templates/server-http`). Mutually exclusive with `--base-ref`.

|`--base-ref`
|
|The base OCI reference for all discovered projects (e.g. `ghcr.io/arconia-io/arconia-project-templates`). Discovers and pushes all projects found as direct subdirectories of `--path`. Mutually exclusive with `--ref`.

|`--path`
|_(current directory)_
|Path to the project directory, or with `--base-ref`: the parent directory to discover projects in.

|`--tag`
|`latest`
|The version tag (e.g. `latest`, semantic version number, commit sha).

|`--annotation`
|
|Additional annotations in key=value format (e.g. `--annotation org.opencontainers.image.vendor=arconia`). Can be specified multiple times.

|`--output-report`
|`project-push-report.json`
|Write a publish report file. Defaults to `project-push-report.json` when specified without a path. The report can be used for downstream operations such as collection pushing (`arconia project collection push --from-report`), artifact signing (e.g. cosign), or SLSA provenance attestation.

|`--registry-insecure`
|`false`
|Call the OCI registry over HTTP instead of HTTPS.

|`--registry-skip-tls-verify`
|`false`
|Allow TLS connections without validating certificates.

|`--verbose` or `-v`
|`false`
|Include verbose output.

|`--help` or `-h`
|
|Display help information for the command.
|===
