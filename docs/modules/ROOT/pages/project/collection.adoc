= `arconia project collection`

The `arconia project collection` command manages collections of projects. Collections are OCI indexes that group available projects and their locations, making them easy to discover and share.

== Commands

=== `add`

Register a project collection with the Arconia CLI configuration and fetch its contents into the local cache.

[source,shell]
----
arconia project collection add \
  --name arconia-project-templates \
  --ref ghcr.io/arconia-io/arconia-project-templates/collection
----

**Options**

[cols="1,1,3"]
|===
|Option |Default |Description

|`--name` (required)
|
|The name of the Java project collection (e.g. `arconia-project-templates`).

|`--ref` (required)
|
|The OCI artifact reference for the collection (e.g. `ghcr.io/arconia-io/arconia-project-templates/collection`).

|`--registry-insecure`
|`false`
|Call the OCI registry over HTTP instead of HTTPS.

|`--registry-skip-tls-verify`
|`false`
|Allow TLS connections without validating certificates.

|`--verbose` or `-v`
|`false`
|Include verbose output.

|`--help` or `-h`
|
|Display help information for the command.
|===

=== `list`

List available projects in a collection. Displays each project's name, description, type, and labels in a table format grouped by collection.

[source,shell]
----
arconia project collection list
arconia project collection list --name arconia-project-templates
----

**Options**

[cols="1,1,3"]
|===
|Option |Default |Description

|`--name`
|
|The name of the registered collection for which to list the projects. If omitted, lists projects from all registered collections.

|`--registry-insecure`
|`false`
|Call the OCI registry over HTTP instead of HTTPS.

|`--registry-skip-tls-verify`
|`false`
|Allow TLS connections without validating certificates.

|`--verbose` or `-v`
|`false`
|Include verbose output.

|`--help` or `-h`
|
|Display help information for the command.
|===

=== `update`

Update a registered collection or all registered collections. For each collection, checks whether a newer semver tag is available in the OCI registry. If a newer version is found, the stored reference is updated to the latest tag and the cache is refreshed. Otherwise, the existing reference is re-fetched to refresh the cache and digest.

[source,shell]
----
arconia project collection update
arconia project collection update --name arconia-project-templates
----

**Options**

[cols="1,1,3"]
|===
|Option |Default |Description

|`--name`
|
|The name of a specific Java project collection to update. If omitted, updates all registered collections.

|`--registry-insecure`
|`false`
|Call the OCI registry over HTTP instead of HTTPS.

|`--registry-skip-tls-verify`
|`false`
|Allow TLS connections without validating certificates.

|`--verbose` or `-v`
|`false`
|Include verbose output.

|`--help` or `-h`
|
|Display help information for the command.
|===

=== `push`

Publish a project collection as an OCI artifact. At least one of `--from-report` or `--project` must be provided.

[source,shell]
----
arconia project collection push \
  --name arconia-project-templates \
  --ref ghcr.io/arconia-io/arconia-project-templates/collection \
  --tag 1.2.3 \
  --project ghcr.io/arconia-io/arconia-project-templates/my-project:1.0.0 \
  --project ghcr.io/arconia-io/arconia-project-templates/other-project:1.4.2

arconia project collection push \
  --name arconia-project-templates \
  --ref ghcr.io/arconia-io/arconia-project-templates/collection \
  --from-report project-push-report.json
----

**Options**

[cols="1,1,3"]
|===
|Option |Default |Description

|`--ref` (required)
|
|The full OCI artifact reference for the project collection (e.g. `ghcr.io/arconia-io/arconia-project-templates/collection`).

|`--name` (required)
|
|The collection identifier (e.g. `arconia-project-templates`).

|`--description`
|`A collection of Java projects published as OCI Artifacts and managed by the Arconia CLI.`
|A short description of the collection.

|`--tag`
|`latest`
|The version tag (e.g. `latest`, semantic version number, commit sha).

|`--from-report`
|`project-push-report.json`
|Path to a push report file (from `arconia project push --output-report`). Defaults to `project-push-report.json` when specified without a path.

|`--project`
|
|Explicit project OCI references to include (e.g. `--project ghcr.io/arconia-io/arconia-project-templates/project:1.0.0`). Can be specified multiple times.

|`--annotation`
|
|Additional annotations in key=value format (e.g. `--annotation org.opencontainers.image.vendor=arconia`). Can be specified multiple times.

|`--output-report`
|`project-collection-push-report.json`
|Write a publish report file. Defaults to `project-collection-push-report.json` when specified without a path. The report can be used for downstream operations such as artifact signing (e.g. cosign) or SLSA provenance attestation.

|`--registry-insecure`
|`false`
|Call the OCI registry over HTTP instead of HTTPS.

|`--registry-skip-tls-verify`
|`false`
|Allow TLS connections without validating certificates.

|`--verbose` or `-v`
|`false`
|Include verbose output.

|`--help` or `-h`
|
|Display help information for the command.
|===
