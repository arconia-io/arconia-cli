= Arconia CLI

[.hero]
Welcome to Arconia CLI! This documentation will help you get started with the tool and explore its features.

== What is Arconia CLI?

Arconia CLI is the companion command-line tool for the https://spring.io/projects/spring-boot[Spring Boot] and https://docs.arconia.io/arconia/latest/index.html[Arconia Framework] projects. It provides intuitive commands that streamline the entire development lifecycle: running applications in development mode with automatic service provisioning, building and testing, packaging as container images, and upgrading to newer versions of Spring Boot, Spring AI, and the Arconia Framework.

NOTE: The Arconia CLI is currently in active development. We're working hard to improve it and appreciate your patience as we continue to refine the tool. Feel free to try it out and share your feedback!

== Quick Start

Install the Arconia CLI using one of the methods described in xref:installation.adoc[Installation].

To get started, view the available commands and options by running:

[source,shell]
----
arconia
----

You'll see a summary of the main commands and options available in the CLI.

[source,shell]
----
Usage: arconia [-hV] [COMMAND]
Options:
  -h, --help      Show this help message and exit.
  -V, --version   Print version information and exit.

Commands:
  dev              Run a Spring Boot application in dev mode.
  build            Build a Java application.
  test             Run tests for a Java application.
  image            Build and manage container images for Spring Boot applications.
  update, upgrade  Update project dependencies and build tool versions.
  rewrite          Discover and run OpenRewrite recipes to migrate or refactor a Java application.
  project          Create and manage Java projects
  skills           Install and manage agent skills.
  help             Display help information about the specified command.
  version          Display version information about the Arconia CLI.
----

== Commands

=== Development

These commands wrap your build tool (Maven or Gradle) to provide a consistent, tool-agnostic developer experience:

* **xref:development/dev.adoc[arconia dev]** — Runs your application in development mode, activating the Arconia Framework's `dev` Spring profile and automatically starting any declared https://docs.arconia.io/arconia/latest/dev-services/[Arconia Dev Services] (databases, message brokers, AI inference servers, etc.) as containers. No boilerplate code required.
* **xref:development/build.adoc[arconia build]** — Compiles, tests, and packages your application as a JAR or a GraalVM native executable.
* **xref:development/test.adoc[arconia test]** — Runs your test suite, with Arconia Dev Services automatically provisioned for integration tests.

=== Container Images

* **xref:image/buildpacks.adoc[arconia image build buildpacks]** — Builds a container image using Cloud Native Buildpacks and Paketo, without needing a Dockerfile.
* **xref:image/dockerfile.adoc[arconia image build dockerfile]** — Builds a container image using a Dockerfile.

=== Updates

These commands use https://docs.openrewrite.org/[OpenRewrite] recipes to automatically migrate your project with a single command:

* **xref:update/spring-boot.adoc[arconia update spring-boot]** — Upgrades to a newer Spring Boot version.
* **xref:update/spring-ai.adoc[arconia update spring-ai]** — Upgrades to a newer Spring AI version.
* **xref:update/framework.adoc[arconia update framework]** — Upgrades to a newer Arconia Framework version.
* **xref:update/gradle.adoc[arconia update gradle]** — Upgrades the Gradle wrapper.
* **xref:update/maven.adoc[arconia update maven]** — Upgrades the Maven wrapper.

=== Migration

* **xref:migration/rewrite-run.adoc[arconia rewrite run]** — Runs any OpenRewrite recipe for custom migration or refactoring tasks beyond the built-in `update` commands.
* **xref:migration/rewrite-discover.adoc[arconia rewrite discover]** — Discovers available OpenRewrite recipes.

=== Projects

* **xref:project/project.adoc[arconia project create]** — Creates a new project from a template.
* **xref:project/project.adoc[arconia project push]** — Publishes a project (or a batch of projects) as OCI artifact(s).
* **xref:project/collection.adoc[arconia project collection]** — Manages collections of projects distributed as OCI artifacts.
