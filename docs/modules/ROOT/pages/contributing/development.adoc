= Development

This page describes how to build, test, and run the Arconia CLI from source.

For contribution guidelines (workflow, commit conventions, DCO), see the https://github.com/arconia-io/arconia-cli/blob/main/CONTRIBUTING.md[Contributing Guide].

== Prerequisites

* Java 25 or later with https://www.graalvm.org/[GraalVM] native compilation support.
* A container runtime compatible with https://www.testcontainers.org/[Testcontainers] (e.g., https://podman-desktop.io[Podman Desktop], https://www.docker.com/products/docker-desktop/[Docker Desktop]).

== Building the CLI

Build on the JVM (compile, test, and package):

[source,shell]
----
./gradlew build
----

Build as a GraalVM native executable:

[source,shell]
----
./gradlew nativeBuild
----

The native executable is generated at `build/native/nativeCompile/arconia`.

== Running Tests

Run all tests on the JVM:

[source,shell]
----
./gradlew test
----

Run all tests in native mode:

[source,shell]
----
./gradlew nativeTest
----

Run a specific test class:

[source,shell]
----
./gradlew test --tests "io.arconia.cli.commands.VersionCommandTests"
----

== Installing the CLI

You can run the CLI directly from its build location or install it system-wide. To install globally on macOS or Linux:

[source,shell]
----
sudo cp build/native/nativeCompile/arconia /usr/local/bin/arconia
----

== Running the CLI

If installed globally:

[source,shell]
----
arconia help
----

Otherwise, use the full path:

[source,shell]
----
./build/native/nativeCompile/arconia help
----
